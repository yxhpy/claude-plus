# 自动设计审查技能

当 Claude 检测到用户需要 UI/UX 设计方面的帮助时，自动调用此技能。

## 触发场景

- 用户询问"这个设计怎么样？"
- 用户请求"帮我看看这个界面"
- 用户提到"UI 有什么问题"
- 用户询问"如何改进用户体验"
- 用户上传了设计稿或截图

## 执行流程

1. **识别审查目标**
   - 从对话上下文中识别要审查的内容
   - 确定是代码、截图还是描述
   - 使用 Read 工具读取相关文件（如果是代码）

2. **使用 Gemini CLI 进行审查**

   构建设计审查提示词并调用 gemini CLI：

   ```bash
   gemini -p "作为资深 UI/UX 设计师，审查以下内容：

   [代码/设计描述]

   从以下角度进行专业审查：
   - 可访问性 (Accessibility): ARIA 标签、键盘导航、屏幕阅读器支持
   - 用户体验 (UX): 交互流程、反馈机制、错误处理
   - 视觉设计 (UI): 布局、颜色、字体、间距、一致性
   - 响应式设计: 移动端适配、断点设计
   - 性能: 渲染性能、动画流畅度

   请提供：
   1. 优点列表
   2. 问题和改进建议（包含代码示例）
   3. 各维度评分（1-10分）
   4. 总结和优先改进建议"
   ```

3. **一次性输出审查报告**
   - 直接显示 Gemini 的审查结果
   - 格式化输出，便于阅读
   - 无需多次验证

## 示例对话

**用户**: "帮我看看这个登录表单的设计"

**Claude**: 我来对这个登录表单进行专业的 UI/UX 审查。

[使用 Read 工具读取表单代码]
[使用 gemini CLI 进行分析]

🎨 设计审查报告：

✅ 优点：
- 布局清晰，视觉层次分明
- 使用了合适的表单验证

⚠️ 改进建议：

2. 用户体验：密码输入框应该有显示/隐藏切换
3. 视觉设计：错误提示颜色对比度不足

[详细建议...]

- ✅ 一次性完成审查，无重复验证
- ✅ 更快的响应速度
- ✅ 更可靠的执行
